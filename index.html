Markovian Tweet Generator

This project works by consuming a text file that has some sample text and generates meaningful sentences a.k.a tweets with <145 characters. The different components that were designed as part of the tweet generator included the following-

  • The File Handler- For taking in the file, opening it and parsing through the sample text.

  • The Hash Map which performed the following- -Parses through a given string to build a word character by character. -Generates the key (word) and value(frequency of words) pairs for a given sample text. -Generates a map of the words based on the key and value. -Takes care of all conditions and exceptions while populating the map.

  • Tweet Generator which did the following- -Take in the file with the sample text and creates the mapping for the words in this text as described in the Hash Map
    block.

    • This block creates a tweet based on the likelihood of words from the map.
    • The tweet generated will be less than 145 characters, where, based on a random value an iteration through the list of values takes place. The random value is generated based on the the total frequencies in the list of values produced in Hash Map. This iteration will look for values whose frequency, added to the previous frequencies hits the threshold of the random value.
